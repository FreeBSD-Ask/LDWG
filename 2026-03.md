## 进行中

### 电源

#### S4 休眠

在社区成员 Warner Losch 和 John Baldwin 的参与下，S4 休眠状态保存的最终实现设计现已取得重大进展。这将使最终实现更加高效，成功几率也更高。

我们正在继续试验系统启动方法并进行调试。此功能已有一个工作分支。

GitHub issue：[#29](https://github.com/FreeBSDFoundation/proj-laptop/issues/29)

#### 功耗管理工具

我们现已实现完整工作流程，可将 acpi 和 rapl 中的电池与电源使用数据导入到 Perfetto 中。我们正在研究基准测试标准，以开发示例问题，记录如何使用 perfetto 分析电源问题。

在开发过程中，我们预计会发现一些需要改进的地方以及使这些示例更实用所需的缺失功能。

GitHub issue：[#6](https://github.com/FreeBSDFoundation/proj-laptop/issues/6)

#### 性能管理工具

这项功能称为 CPPC（协作处理器性能控制），将有助于提升 FreeBSD 在配备相关处理器的笔记本电脑上的性能和效率。

我们正在努力改进 Intel 相关功能，使其与 AMD 功能保持一致。同时，AMD 实现也进行了一些重构。

GitHub issue：[#108](https://github.com/FreeBSDFoundation/proj-laptop/issues/108)


### 安装

#### KDE 桌面安装方案

该功能仍按计划在 15.1 版本中推出。本月进展包括优化用户界面的错误和信息提示，以及改进 GPU 自动检测和驱动选择菜单。

GitHub issue：[#25](https://github.com/FreeBSDFoundation/proj-laptop/issues/25)

### Wi-Fi

#### MediaTek Wi-Fi 卡支持

我们继续努力改进 MediaTek 无线支持，通过优化共享依赖项并确保驱动程序能从持续的修复中受益，使其与其他无线卡支持保持一致。

MediaTek Wi-Fi 卡的 USB 支持正在开发中。


GitHub issue：[#66](https://github.com/FreeBSDFoundation/proj-laptop/issues/66)

#### Realtek Wi-Fi 卡支持动态

事实证明，Realtek 相关工作还需要一些额外努力（我们原以为上月已完成），目前工作重点是修复连接问题、改进网络发现方式，以及处理部分卡的特殊问题以提高稳定性。

GitHub issue：[#99](https://github.com/FreeBSDFoundation/proj-laptop/issues/99)

#### Wi-Fi 6

我们目前正在努力让更多支持 Wi‑Fi 6（11ax 和 11be）的驱动程序（除 Intel 外，还包括 Realtek 和 MediaTek）正常工作，之后将恢复堆栈工作（原生 nat80211 和 LinuxKPI）。

GitHub issue：[34](https://github.com/FreeBSDFoundation/proj-laptop/issues/34)
