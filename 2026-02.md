## 正在进行

### 电源

#### S4 休眠

将计算机状态保存到 S4 映像的开发工作现在已经足够成熟，可以开始开发唤醒功能，而该工作已在二月开始。

GitHub issue：[ #29 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/29)

#### 电源消耗管理工具

目前正在进行的工作是使 Perfetto Port 能够在笔记本电脑上运行时报告 FreeBSD 关键组件的电源消耗。正在创建探针，以检测通过 Type-C 连接的设备的功耗，并检测电池和内核事件，这些信息结合起来能用于构建对电力使用区域的理解。

这项工作正在扩展以覆盖更多电源接口，从而能够在 Intel 和 AMD 硬件上提供系统电源消耗的更完整图景。

GitHub issue：[ #6 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/6)

#### 性能管理工具

目前正在进行的工作是启用 AMD 处理器的一项原生功能，该功能能让用户使用手动模式或自动模式来控制笔记本电脑性能的某些方面。

这一功能称为 CPPC（Collaborative Processor Performance Control），它将有助于提升 FreeBSD 在搭载相关处理器的笔记本电脑上的性能和效率。

GitHub issue：[ #108 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/108)

#### 低功耗状态调试工具

多种 USB 调试机制的工作仍在继续，这些机制可以在没有可用图形驱动程序（低功耗状态所需）的情况下工作。

在所有 FreeBSD 版本中，针对 2025 年及之后的 Framework 笔记本电脑，已经提供了可用的 USB 串口调试接口以及使用 SoC 的调试接口。FreeBSD 16 将提供 xhci 调试，并且可以从正在进行的代码审查中进行评估。

目前我们正在记录这些功能，并将在适当时候更新相关材料。

GitHub issues：[ #38 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/38) 和 [ #90 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/90)

### Wi-Fi

#### 对 MediaTek Wi-Fi 卡的支持

已导入来自制造商的最新驱动更新，但仍有一些依赖项需要更新，以便一切能够按预期工作。

对 MediaTek Wi-Fi 卡的 USB 支持正在进行中。

GitHub issue：[ #66 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/66)

#### WiFi 6 支持

工作正在进行中，其中大部分工作集中在更新 net80211，这是开发 WiFi 6 功能所需的底层组件。二月开始调查当 net80211 发生更改时哪些 Port 会出现问题，以便研究并解决这些问题。

GitHub issue：[ 34 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/34)

### 显卡

#### 移植 Linux 6.11 显卡驱动

Linux 6.11 显卡驱动的移植工作目前大约完成了 50%。

GitHub issue：[ #52 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/52)

### 系统管理

#### 让 pkgbasify 更高效

改进 pkgbasify 工作效率的任务仍在继续。

GitHub issue：[ #91 ](https://github.com/FreeBSDFoundation/proj-laptop/issues/91)

### 集成测试

#### 针对笔记本项目功能的整体测试

针对已提供（已完成）的 *Laptop Project* 功能的测试现在已经完成，测试对象包括 [ Framework 13 7040 Ryzen 系统 ](https://github.com/FreeBSDFoundation/proj-laptop/blob/main/supported/laptops.md)、一台 Framework 桌面机器以及一台 Lenovo Yoga 11e。

计划在三月公布一个简单的社区贡献者测试工作流程，以便用户可以提交标准化测试脚本的结果。

可供查询的公开笔记本兼容性参考的概念验证也在推进中，供用户使用。我们希望整合并在现有类似社区资源的基础上进行构建，从而制作一个可持续且改进的页面。
